Hvorfor kotlin?

  • Kotlin er foretrukket av mange utviklere
  • Fungerer med eksisterende biblioteker, selv om det er unntak (feks Threads/coroutines)
  • Migrering av Java applikasjoner er enkelt
  • Kotlin er et godt verktøy for DSLer
Anders ASK ABOUT QUESTIONS Kotlin is an official language in the NTA, developers can choose between kotlin and java for new projects Spoiler, they choose kotlin ;) Converting Java applications to Kotlin, an example is an application for database management which has been continuously converted to kotlin feature-by-feature

Kotlin language features

Dataklasser, navngitte og default argumenter


data class SomeClass(
    val someValue: String = "two"
)

val obj = SomeClass("one")    		// one
val obk = SomeClass(someValue = "one")  // one
val obl = SomeClass()                   // two

Extension functions


fun String.doTheThing(): Boolean = "".isBlank()

val falseV = "lol".doTheThing()
val truthV = "".doTheThing()

Hva har vi lært?

  • Kotlin er veldig godt tatt i mot fra utviklerne
  • Funksjonalitet som extension functions og null safety hjelper oss unngå bugs/feil ved å tilpasse eksisterende API
  • Noen biblioteker fungerer ikke med coroutines, noe som ikke alltid er opplagt
  • Thread.local (used by Spring security, log4j osv) fungerer ikke bra med coroutines
